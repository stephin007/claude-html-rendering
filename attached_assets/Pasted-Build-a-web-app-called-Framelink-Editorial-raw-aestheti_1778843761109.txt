Build a web app called Framelink. Editorial, raw aesthetic — 
like a developer zine meets a serious tool. Monospace heavy, 
stark contrast, brutalist but functional. NOT generic AI slop. 
NO purple gradients. NO rounded hero sections. NO Inter font.

Browse https://21st.dev and use components that match a raw, 
editorial, high-contrast dark aesthetic.

WHAT FRAMELINK DOES:
A user uploads an HTML file. It renders live in the browser 
inside an iframe. A shareable link is generated that anyone 
can open without logging in. Reviewers click anywhere on the 
rendered design to drop a numbered comment bubble (like Figma 
comments). They type a comment, hit save, bubble stays pinned. 
Owner sees all comments in a right side panel. 
"Copy all for Claude" button copies unresolved comments 
as a formatted list.

FILE STRUCTURE:
index.html — upload page
view.html — review page (what the shareable link opens)
storage.js — all Supabase logic
schema.sql — database schema
style.css — global styles

INDEX.HTML:
- Top left: wordmark "framelink" in monospace, all lowercase
- No hero. No tagline. No marketing copy.
- Center of screen: a raw upload zone. Thick dashed border. 
  Monospace label "DROP HTML FILE" in caps. Nothing else.
- Below upload zone: small monospace text showing recent 
  uploads (just file names, no thumbnails)
- When file is uploaded → save to Supabase → 
  redirect to view.html?id=PROTOTYPE_ID

VIEW.HTML:
- Top bar: "framelink" wordmark left, file name center 
  in monospace, comment mode toggle right
- Comment mode toggle: raw button, no rounded pill. 
  When active shows "// COMMENTING" in accent color
- HTML file renders inside an iframe, full left side, 
  white background (it's someone's design)
- Transparent overlay div on top of iframe, 
  captures clicks when comment mode is on
- On click: small popup at click position. 
  Raw textarea, no border radius. Save/Cancel in monospace.
- Numbered comment bubbles on overlay at x/y percentage positions
- Bubble style: square with a corner notch, not a circle. 
  Accent color background, white number in monospace.
- Right panel 280px: header "COMMENTS" in monospace caps, 
  count next to it
- Comment cards: no border radius, left border accent line only, 
  number + text + resolve button
- Bottom of panel: "→ COPY ALL FOR CLAUDE" button, 
  full width, monospace, no border radius

STORAGE.JS — Supabase functions:
- savePrototype(htmlContent, fileName) → returns uuid
- getPrototype(id) → returns htmlContent, fileName
- saveComment(prototypeId, x, y, text) → saves comment
- getComments(prototypeId) → returns all comments
- resolveComment(commentId) → marks resolved

Use Supabase JS client via CDN.

SHAREABLE URL: view.html?id=PROTOTYPE_ID

SUPABASE PLACEHOLDERS:
const SUPABASE_URL = 'YOUR_SUPABASE_URL'
const SUPABASE_ANON_KEY = 'YOUR_SUPABASE_ANON_KEY'

SCHEMA.SQL — two tables:
prototypes: id (uuid primary key default gen_random_uuid()), 
html_content (text), file_name (text), 
created_at (timestamp default now())

comments: id (uuid primary key default gen_random_uuid()), 
prototype_id (uuid references prototypes(id)), 
x (float), y (float), text (text), 
resolved (boolean default false), 
created_at (timestamp default now())

DESIGN SYSTEM — non negotiable:
- Background: #0a0a0a
- Surface: #111111
- Border: #222222
- Accent: #5b7cf6
- Text primary: #f0f0f0
- Text muted: #444444
- Font: "DM Mono" from Google Fonts everywhere. 
  Nothing else. Not even for body text.
- Zero border radius anywhere except comment popup (2px max)
- No gradients. No shadows. No blur.
- Borders are 1px solid #222222 or 1px solid accent
- Spacing is generous — lots of whitespace between elements
- Every interactive element has a clear hover state: 
  border color switches to accent color
- Active states: background switches to accent, text to #0a0a0a

INTERACTIONS:
- Comment overlay click coordinates stored as percentages 
  not pixels — so they render correctly at any screen size
- When comment mode is OFF the overlay is pointer-events none
- Clicking a comment card in the panel highlights 
  the corresponding bubble on the design (add a ring)
- Resolved comments collapse in the panel, 
  greyed out bubble on design
- Copy all for Claude formats as:
  "Feedback on [filename] — [n] comments:
  #1 — [comment text]
  #2 — [comment text]"

REPLIT SPECIFIC:
- All files served statically
- No build step, no npm, no node
- Everything via CDN only
- Make sure view.html can read the ?id= param on load, 
  fetch the prototype HTML from Supabase, 
  render it in the iframe, and load existing comments
